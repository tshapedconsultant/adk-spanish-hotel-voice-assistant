Flujo webhook (demo e integraciones)

Petición típica con session_id UUID v4, rate limit y clave opcional.

sequenceDiagram
  participant U as Cliente (demo / API)
  participant F as Flask /webhook/trigger
  participant L as Rate limiter
  participant A as GeminiAgent
  participant S as Session store
  participant G as Gemini API
  participant C as Callbacks

  U->>F: POST JSON text, session_id?
  F->>F: Validar WEBHOOK_API_KEY
  F->>L: allow(remote_addr)
  alt límite excedido
    F-->>U: 429 Too many requests
  end
  F->>F: Validar tipos y MAX_TEXT_CHARS
  F->>A: generate(text, session_id)
  A->>S: cargar / crear sesión
  A->>G: chat + routing (intención / entidades)
  G-->>A: respuesta + routing
  A->>S: persistir historial
  opt intent booking_request
    A->>C: on_booking_request(payload)
    C-->>PMS: HTTP opcional
  end
  A-->>F: reply, session_id
  F-->>U: 200 JSON

Enrutamiento de intención por turno

Structured routing por defecto; fallback por regex si el LLM de routing falla o está desactivado.
